# 德州扑克模拟练习器

一个面向新手的德州扑克模拟练习器 MVP。目标是让初学者先进行尽可能纯粹的对局，再在手牌结束后通过复盘和长期数据统计发现自己的问题。

当前版本是纯静态前端，无需构建工具。由于脚本使用 ES module，请通过本地静态服务访问。

## 交接文档

后续开发优先阅读 [docs/HANDOFF.md](docs/HANDOFF.md)。这份文档记录当前项目状态、关键文件、测试方式、已知限制和下一步建议，用于新窗口或新协作者快速接续。

## 功能

- 6 人桌 No-Limit Hold'em 练习流程
- 多种常规桌 AI 对手风格：标准 TAG、常规 Reg、紧手、跟注、松凶
- 对局中不展示策略提示，减少干扰
- 单手结束后进入独立复盘视图
- 历史手牌库：每手牌结束自动入库，可筛选并选择任意手牌复盘
- 牌桌配置：HU、6-max、9-max、短码、深筹码和自定义盲注/买入
- 复盘展示每个关键决策点的简化策略建议
- 行动按钮：弃牌、看牌 / 跟注、加注
- 手牌记录和 JSON 导出
- 长期数据统计：VPIP、PFR、3Bet、AF、WTSD、W$SD、WWSF、CBet、Fold to CBet、bb/100
- 基于数据的初步问题诊断
- 后续接入 TexasSolver / postflop solver 的复盘接口预留

## 本地运行

启动本地静态服务：

```powershell
python -m http.server 4173 --bind 127.0.0.1
```

然后打开：

```text
http://127.0.0.1:4173/
```

## 规则测试

项目把关键德州扑克规则放在 `rules.js`，并用 `engine.js` 提供完整手牌状态机。当前测试无第三方依赖：

```powershell
npm test
```

当前测试覆盖：

- 行动权过滤
- 下注轮是否完成
- 最小加注目标
- 完整加注是否重新打开行动权
- all-in 不足额加注不重新打开行动权
- all-in 后是否自动发完公共牌并摊牌
- A2345 顺子、同花、葫芦、踢脚比较
- 主池 / 边池分配
- 平分底池
- 弃牌玩家贡献进入底池但不能赢池
- 按钮位、盲注和翻后首个行动位
- heads-up 盲注和行动顺序
- 非法筹码提交 / 非活动玩家下注保护
- 多层边池
- 总筹码守恒
- 随机边池压力测试
- 完整手牌从盲注到河牌摊牌
- 翻前 limp 后大盲行动权
- 短码 all-in 不重新打开行动权
- 越位行动 / 非法 check 拒绝
- 合法动作、跟注额、最小/最大加注边界
- 结构化 hand history
- 按行动当时状态抽取复盘决策点
- solver-facing spot JSON 雏形

## 项目定位

这个项目目前不是完整 GTO 求解器。AI 和复盘建议使用的是简化策略模型，用来让对局更接近正常牌桌，并帮助新手理解“为什么这个行动看起来更合理”，不是高精度 solver 策略表。

后续计划把练习器拆成三层：

1. 牌局引擎：使用 PokerKit 或后端服务替换前端内置牌局逻辑。
2. AI 对手：逐步加入范围表、权益估算、强化学习或预训练策略的不同对手。
3. GTO 复盘：将牌局节点转换为 TexasSolver / postflop solver 输入，展示策略频率、EV 差距和更自然的新手解释。
4. 数据分析：按位置、街次、行动类型拆分统计，形成更具体的 leak report。
